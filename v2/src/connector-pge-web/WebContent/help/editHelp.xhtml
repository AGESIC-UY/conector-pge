Nuevo Conector

Todos los datos del conector son requeridos.

En el ingreso de datos si no se ingresa uno de los campos requeridos al momento de apretar el botón salvar, se volverá a mostrar la misma página indicando cual es el campo en el cual no se ingresaron datos. Si se da este caso y se había seleccionado un archivo, para cualquiera de los campos Wsdl, Keystore Organismo, Keystore SSL o Trustore SSL , este se perderá y se tendrá que volver a seleccionar el o los archivos.

La combinación "Nombre" - "Tipo" tiene que ser única en el sistema, así como también la combinación "Path" - "Tipo".

Descripción de los datos

A continuación se detalla el significado de cada dato y el impacto de estos hacia el sistema.

  • Nombre: Nombre del Servicio. Se usa como identificación interna del servicio y junto con el Tipo deben ser únicos.
  • Tipo : El tipo de servicio identifica si este es un servicio de Test o de Producción.
  • Descripción : Información descriptiva del servicio sin uso operativo.
  • Path : Es el path relativo en donde queda publicado el servicio. Por ejemplo, si el path es /censo/CensoINEService entonces si el tipo es "Test" el servicio queda publicado en http://server_host:9800/censo/CensoINEService y si el tipo es "Producción" el servicio queda publicado en http://server_host:9700/censo/CensoINEService
  • wsa:To : Este dato se agrega como Header del mensaje SOAP. Es un endpoint lógico e indica cual es el Web Service que se quiere invocar a través de la plataforma.
  • Username : Usuario dentro del organismo que desea acceder al servicio final, este dato es utilizado para la firma del mensaje SOAP.
  • Organismo : Nombre que representa al organismo consumidor, este dato es utilizado también para la firma del mensaje SOAP
  • Tipo de Token : Dato utilizado para la firma del SOAP.
  • Ingresar credenciales de Username Token : Habilita el ingreso de usuario y contraseña para Username Token.
  • Habilitar Configuración Local : Permite al usuario definir los keystores y truststores que utiliza el conector. En caso de no seleccionar este control, se utiliza los certificados globales.
  • Habilitar cache de Tokens : Permite cachear los tokens SAML pedidos a la plataforma, de manera de no volver a pedir tokens mientras se tenga uno válido.Esto se puede habilitar solo cuando la hora este sincronizada con el servidor NTP.
  • Alias del Keystore Organismo (Config Local) : Alias en donde se encuentra el certificado que se quiere utilizar para la firma del SOAP. Este alias corresponde al "Keystore Organismo" (En el caso que sea un conector con configuración local habilitada)
  • Password Keystore Organismo (Config Local) : Password del "Keystore Organismo" (En el caso que sea un conector con configuración local habilitada)
  • Password Keystore SSL (Config Local) : Password utilizada para acceder al certificado del keystore para el acceso por https. (En el caso que sea un conector con configuración local habilitada)
  • Password Truststore (Config Local) : Password utilizada para acceder al certificado del trustore para el acceso por https. (En el caso que sea un conector con configuración local habilitada)
  • Keystore Organismo (Config Local) : Keystore del Organismo utilizado para la firma del SOAP (En el caso que sea un conector con configuración local habilitada)
  • Keystore SSL (Config Local) : Keystore utilizado para la comunicación HTTPS (En el caso que sea un conector con configuración local habilitada)
  • Truststore SSL (Config Local) : Truststore utilizado para la comunicación HTTPS (En el caso que sea un conector con configuración local habilitada)
  • WSDL : Archivo WSDL en donde está especificado el endpoint al cual se va a invocar al servicio y las operaciones que se pueden invocar. Este campo acepta archivos wsdl, así como también archivos zip que contengan el wsdl junto a sus dependencias (otros archivos wsdl o xsd).
  • Tag : Atributo para filtrar el conector

Al lado del campo de entrada para seleccionar el archivo WSDL, hay un botón de "refresh". Ese hace que se cargue el archivo WSDL y se actualicen las operaciones del conector con las operaciones especificadas en dicho WSDL. Dicho archivo no es guardado hasta que no se guarden los cambios.

Edición

En la edición se muestran los datos del conector, permitiendo su modificación (a excepción del "Nombre y "Tipo" que no pueden ser modificados luego de creado el servicio). Además se permite sobrescribir un archivo (ya sea un keystore, truststore, wsdl) al seleccionar uno nuevo.

La pantalla también permite borrar el conector (utilizando el botón de Borrar) y pasar el conector a producción (botón Pasar a Producción). Esta última funcionalidad se encuentra habilitada para los conectores de tipo Test, y lo que hace es crear un conector en Producción copiando todos los datos del conector de Test.

Los conectores de Producción no tienen habilitado el botón de Pasar a Producción, pero en caso que hayan sido creados a partir de un conector de Testeo, se muestra el botón de Test Asociado, de manera de poder navegar desde el conector de Producción a Testeo.
